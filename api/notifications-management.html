Retrieve Notification Service Status

POST
https://uat-api.enza.cloud/TX/v1
/notification/status

Retrieve the active notification channels (SMS, Email) for an existing customer

Retrieve Notification Service Status Headers

  • x-api-keystringrequired

    API key for authentication

  • Content-Typestringrequired

    Content type of the request body

    Default: application/json

Retrieve Notification Service Status Request Body

  • nationalIdstring

    Conditionally Required: National ID (required if passport and identityId not provided)

    Example: 321321
  • passportstring

    Conditionally Required: Passport number (required if nationalId and identityId not provided)

    Example: AB1234567
  • identityIdstring

    Conditionally Required: Identity ID (required if nationalId and passport not provided)

    Example: ID123456

Retrieve Notification Service Status Responses

Notification status retrieved successfully

  • codestringrequired
    Example: SUCCESS_NOTIFICATION_STATUS_RETRIEVED
  • paramsobjectrequired

Manage Notification Service

POST
https://uat-api.enza.cloud/TX/v1
/notification/manage

Enable notification service for a customer by specifying which service to be active.

How It Works:

  1. Customer Verification: Verifies customer existence using National ID, Passport, or Identity ID
  2. Service Activation: Specifies which notification service to be active
  3. Exclusive Behavior: Activates only what is sent and automatically suspends any other notification service

Note: To check currently active notification services before making changes, refer to the [POST /notification/status] endpoint.

Manage Notification Service Headers

  • x-api-keystringrequired

    API key for authentication

  • Content-Typestringrequired

    Content type of the request body

    Default: application/json

Manage Notification Service Request Body

  • nationalIdstring

    Conditionally Required: National ID (required if passport and identityId not provided)

    Example: 7623835006
  • passportstring

    Conditionally Required: Passport number (required if nationalId and identityId not provided)

    Example: AB1234567
  • identityIdstring

    Conditionally Required: Identity ID (required if nationalId and passport not provided)

    Example: ID123456
  • channelstring

    Required. Notification channel(s) to activate.

    IMPORTANT: Requesting a channel will replace/suspend any existing notification configuration.

    • To activate Email only: send email (this will suspend SMS if active)
    • To activate SMS only: send sms (this will suspend Email if active)
    • To activate BOTH channels: send sms,email (comma-separated, no spaces)
    Example: sms,email
  • statusstringenum

    Optional. Service status: Active or Suspend (default: Active)

    Enum values:
    Active
    Suspend
    Example: Active

Manage Notification Service Responses

Notification service created/updated successfully

  • codestringrequired
    Example: SUCCESS_NOTIFICATION_SERVICE_ENABLED
  • paramsobjectrequired
