Overview

​Enza REST API Documentation

Welcome to the Enza REST API Developer Portal

Our API provides seamless integration for instant issuing services across Africa. This portal offers all the necessary tools, detailed documentation, and API endpoints required to empower your financial operations with real-time card issuance capabilities.

Key Features

  • Instant Card Issuance: Issue physical and virtual cards in real-time
  • Card Management: Complete lifecycle management from creation to closure
  • Transaction Monitoring: Real-time transaction tracking and reporting
  • PIN Management: Secure PIN operations for enhanced security
  • E-Commerce Controls: Manage online transaction capabilities

Getting Started

All API requests require:

  • x-api-key is required for authentication

​Instant Issue Card

POST
https://uat-api.enza.cloud/TX/v1
/instantissuing

Instantly issue a card for a new or existing customer.

This endpoint handles two scenarios:

ScenarioRequestType Value
New customerNewAccountNewCustomer
Existing customerNewAccountExistingCustomer

​Headers

  • x-api-key
    string
    required

    API key for authentication

  • Content-Type
    string
    required

    Content type of the request body

​Request Body

  • RequestType
    string

    Required. Request type:

    • NewAccountNewCustomer: New customer
    • NewAccountExistingCustomer: Existing customer
  • CardRefNum
    string

    Required. Card reference number

  • EmbossedName
    string

    Required. Name to be embossed on the card

  • AccountNum
    string

    Required. Account number to link to the card

  • FirstName
    string

    Required for new customers. Customer's first name

  • LastName
    string

    Required for new customers. Customer's last name

  • NationalId
    string

    Either NationalId or Passport is required. Customer's national ID

  • Passport
    string

    Either NationalId or Passport is required. Customer's passport number

  • MiddleName
    string
    optional

    Optional. Customer's middle name

  • BirthDate
    string
    optional

    Date of birth in YYYY-MM-DD format

  • Email
    string
    optional

    Customer's email address

  • MobilePhone
    string
    optional

    Customer's mobile phone number

  • StreetTitle
    string
    optional

    Customer's street address

  • Zip
    string
    optional

    Customer's postal code

    • params
