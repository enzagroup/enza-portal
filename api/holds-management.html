Holds Management

Endpointhttps://uat-api.enza.cloud/TX/v1

Holds Management

Manage reserved funds and blocked amounts on customer accounts securely. This section enables financial institutions to place, release, and adjust holds to protect transactions, ensure regulatory compliance, and deliver better service to end-users.

Available Operations

EndpointDescription
Create Debit HoldPlace a hold on an account to reserve a specific amount, preventing it from being used until released or modified.
Release HoldRemove a previously placed hold and make the reserved funds available again.
Increase HoldIncrease the reserved amount on an existing hold.
Decrease HoldReduce the reserved amount on an existing hold.

How It Works

  1. Place a hold — Submit the account number and amount to reserve. A confirmation is returned with the hold reference and status.
  2. Adjust if needed — Increase or decrease the hold amount using the hold reference and account number. Updated details are returned.
  3. Release when done — Remove the hold to make funds available again. The system confirms the release.

Create Debit Hold

POST
https://uat-api.enza.cloud/TX/v1
/holds/debit

Place a manual hold amount on a debit account using the Account Number. The hold amount will reduce the available balance and prevent the customer from using the held amount. The hold remains blocked until the defined CloseTime is reached or the hold is released manually.

Create Debit Hold › Headers

x-api-key
​string · required

API key for authentication

Content-Type
​string · required

Content type of the request body

Default: application/json

Create Debit Hold › Request Body

accountNumber
​string

Required. Target account number (contract RID) to place hold on - numeric only, 1-20 digits

Example: 100000000001488
amount
​number

Required. Hold amount - must be positive (minimum 0.01)

Example: 10
ccy
​string

Required. Currency code (ISO 3166-1 numeric, 3 digits, e.g., 840 for USD)

Example: 840
closeTime
​string

Required. Hold expiry date/time in ISO 8601 format without milliseconds. Examples: 2025-02-18T00:00:00 (start of Feb 18), 2025-02-19T23:59:59 (last second of Feb 19)

Example: 2025-02-18T00:00:00

Release Hold

POST
https://uat-api.enza.cloud/TX/v1
/holds/release

Release (close) an existing hold on a debit account using the Contract RID and HoldId. The hold record is closed and the available balance is adjusted accordingly. The hold can no longer be modified or released again.
