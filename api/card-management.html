Load funds instantly onto a card. This operation credits the specified amount to the card's available balance, enabling immediate use for transactions.

Transaction Flow:

  1. Validate card existence and status
  2. Verify source account balance

‚ÄãConvert Virtual Card to Physical

POST
https://uat-api.enza.cloud/TX/v1
/cards/convert-virtual-to-physical

Convert a virtual card to physical by updating its lifecycle phase to "ordered" status, ready for printing and delivery.

Requirements:

  • Card must be in active status
  • Card must be virtual type
  • Card must be eligible for physical conversion

Process:

  1. Validate card existence
  2. Check card is active
  3. Verify card is virtual type
  4. Update lifecycle phase to "ordered"
  5. Queue card for physical production

‚ÄãCreate Virtual Prepaid Card

POST
https://uat-api.enza.cloud/TX/v1
/cards/create-prepaid

Complete end-to-end card issuance with customer onboarding and optional automatic activation.

üìñ How to Determine Request Type

If you don't know the customer's status, follow this workflow:

  1. First, Check Customer Existence using POST /customers/search (see Customer Management ‚Üí Check Customer Existence in the sidebar)
  • Search by nationalId, passport, or identityId
  • This will tell you if the customer already exists in the system
  1. Then, Choose the Correct Request Type:
    • If customer does NOT exist ‚Üí Use requestType: "NewAccountNewCustomer"
    • If customer exists ‚Üí Use requestType: "NewAccountExistingCustomer"

Supports both New and Existing Customers:

  • New Customer (NewAccountNewCustomer): Creates customer profile and issues card
  • Existing Customer (NewAccountExistingCustomer): Links card to existing customer profile

Process Flow:

  1. Customer Verification: Check if customer exists using document ID (NationalId, Passport, or IdentityId)
  2. Customer Creation: Create new customer profile (if new) OR link to existing customer
  3. Card Issuance: Issue the card
  4. Card Activation (Optional): Set autoActivate: true to automatically activate the card (default: false)
  5. Notifications: Set up SMS/Email notifications (if enabled)

Note: By default, card is NOT automatically activated unless autoActivate is set to true
